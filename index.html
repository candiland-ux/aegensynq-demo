Glossary

Plain-English definitions for the terms you'll see across the AegenSynq platform. Hover any underlined term in the app for an inline definition.

Core Concepts

Canonical schemaThe unified data model
A standardized data model where every connected system's records get translated into one common format. Think of it as a universal translator — Jira's "assignee" becomes the same field as Salesforce's "owner" and NetSuite's "PM."
Entity resolutionMatching the same person across systems
Recognizing that "John Smith" in Jira, "J. Smith" in Salesforce, and "johnsmith@northwind.com" in Slack are all the same person — and merging them into one record.
Cross-system conflictWhen two tools disagree
When two connected systems contain contradictory information about the same thing. Example: Jira says a project is "On Track" but NetSuite shows the budget is 80% spent on 60% of work.
MCPModel Context Protocol
The open standard Anthropic introduced (now adopted by OpenAI, Google, and Microsoft) that lets AI systems connect to tools through one common protocol instead of custom integrations for each platform.

Data Quality Terms

DriftNaming inconsistency over time
When the same concept gets recorded with different field names or values over time. Example: one team writes "client," another writes "customer," another writes "account." All mean the same thing — but no system catches the inconsistency.
DeduplicationRemoving duplicate records
Finding and merging records that refer to the same person, project, or vendor. Most companies have 15–30% duplicate records hiding across their tools.
NormalizationStandardizing values
Converting different ways of writing the same thing into one standard form. Example: "WIP," "in-progress," "Active," "Started" all become the canonical value active.
Health scoreOverall data quality grade
A letter grade (A through F) summarizing how clean and consistent your data is across all connected systems. Computed from duplicate rate, naming drift, conflicts, and completeness.

Auto-actionPre-approved routine task
A routine cleanup the platform performs automatically based on rules you've already approved. Example: archiving stale tickets after 90 days of no activity. Every action requires your approval and is logged to an immutable audit trail.
Human-in-the-loopYou stay in control
AegenSynq never moves your data without permission. Every meaningful action — overrides, merges, archives — gets routed to a human for approval. You decide what's automatic and what needs review.
Status overrideReplacing human-entered status with data verdict
When the data clearly contradicts a manually-entered project status, AegenSynq recommends overriding the human entry with the objective verdict from the data. You approve or reject the override.
Confidence scoreHow sure the AI is
A percentage showing how certain AegenSynq is about a recommendation, based on how strongly the evidence supports it. Anything above 90% is typically routine. Below 90% routes for human review.
